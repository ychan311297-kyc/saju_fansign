# 당신의 최애를 찾아보세요 💗

팬사인회·이벤트 콘텐츠용 **사주 기반 진짜 최애 찾기 웹앱**입니다.

사용자의 생년월일과 태어난 시각을 입력하면 등록된 아이돌·배우·가수 중에서 다음 세 가지 결과를 따로 보여줍니다.

- 💥 **설렘 1위** — 가장 빠르게 입덕하고 강하게 몰입할 인물
- 🫶 **안정 1위** — 오래 좋아해도 지치지 않고 애착이 유지될 인물
- 👑 **종합 1위** — 설렘과 안정이 함께 강한 진짜 최애

모든 계산은 브라우저 안에서 실행되며, 사용자가 입력한 생년월일·시간은 외부 서버로 전송되지 않습니다.

## ✨ 주요 기능

| 기능 | 설명 |
|---|---|
| 사주 계산 | 생년월일로 년주·월주·일주·일간·일지 계산 |
| 태어난 시각 | 입력 시 사용자 시주와 시지 궁합 반영 |

| 설렘·안정 분리 | 빠른 입덕과 장기 덕질 가능성을 따로 표시 |
| 운명 보너스 | 특별한 합·보완 조건에 최대 +15점 |
| 최애 케미 타입 | 입덕 직진형, 평생 최애형, 힐링 최애형 등 |
| 카드 등급 | TRUE FAVORITE, DESTINED BIAS, ULTIMATE BIAS |

# 2. 사용 방법

## 입력값

### 생년월일

8자리 숫자로 입력합니다.

```text
19971231
```

### 태어난 시각

선택 입력입니다.

```text
09:55
```

시간을 입력하지 않으면 시주 궁합은 중립점수 60으로 계산합니다.

### 사용자 성별

선택 사항입니다.

| 선택 | 적용 |
|---|---|
| 여성 | 관성(정관·편관)을 소폭 가중 |
| 남성 | 재성(정재·편재)을 소폭 가중 |
| 미선택 | 성별 중립 |

본 프로그램은 연애 상대가 아니라 **최애**를 찾는 콘텐츠이므로 성별 가중치는 순위를 살짝 조정하는 정도로만 사용합니다.

# 3. 점수 계산

기본 종합점수는 8개 항목을 0~100점으로 계산한 뒤 합산합니다.

```text
기본 종합점수
= 최애 십신 × 35%
+ 일지 케미 × 18%
+ 천간합 × 10%
+ 오행 밸런스 × 10%
+ 신강·신약 보완 × 12%
+ 시주 궁합 × 5%
+ 지장간 공명 × 5%
+ 년지·월지 지속성 × 5%
```

| 평가 항목 | 비중 |
|---|---:|
| 💗 최애 십신 | 35% |
| 💞 일지 케미 | 18% |
| ✨ 천간합 | 10% |
| 🌈 오행 밸런스 | 10% |
| 🫶 신강·신약 보완 | 12% |
| 🌙 시주 궁합 | 5% |
| 🌿 지장간 공명 | 5% |
| 🔗 년지·월지 지속성 | 5% |

자세한 점수표는 [`docs/SCORING.md`](docs/SCORING.md)를 참고하세요.

# 4. 결과에 표시되는 내용

각 결과 카드에는 다음 항목이 표시됩니다.

- 순위
- 이름과 그룹
- 상대 일주와 오행
- 최애 십신
- 최애 역할 해석
- 일지 관계
- 시지 관계
- 성별 보정 설명
- 최애 케미 타입
- 운명 보너스 적용 조건
- 설렘 점수
- 안정 점수
- 최종 최애점수
- 8개 세부 평가항목 점수
- 기본 종합점수
- 운명 보너스

결과 상단에는 다음 세 명을 따로 표시합니다.

```text
💥 설렘 1위
🫶 안정 1위
👑 종합 1위
```

# 5. 최애 케미 타입

| 타입 | 대표 조건 |
|---|---|
| 👑 운명 최애형 | 천간합 + 육합 + 운명 보너스 우수 |
| 💎 균형형 최애 | 설렘·안정 모두 높고 점수 차이가 작음 |
| 💥 입덕 직진형 | 상관 + 천간합 |
| 🫶 평생 최애형 | 식신 + 안정점수 90 이상 |
| 🌿 힐링 최애형 | 인성 + 오행 보완 우수 |
| 🔥 애증 최애형 | 편관·상관 + 일지 충 |
| 🌸 친구 같은 최애 | 비견 + 삼합·비화 |
| ✨ 숨은 보석형 | 편인 + 지장간 공명 우수 |
| ⚡ 강렬 입덕형 | 설렘이 안정보다 크게 높음 |
| 💗 균형 잡힌 최애 | 설렘과 안정이 고르게 형성 |

# 6. 카드 등급

| 최종점수 | 등급 |
|---|---|
| 80 미만 | 🌱 천천히 알아갈 최애 |
| 80~89.99 | 💗 오래 볼수록 좋아지는 최애 |
| 90~99.99 | 💖 TRUE FAVORITE |
| 100~106.99 | 💜 DESTINED BIAS |
| 107 이상 | 👑 ULTIMATE BIAS |

운명 보너스가 붙으면 최종점수는 100점을 넘을 수 있습니다.

# 7. 인물 데이터 추가

인물 데이터는 다음 파일에서 관리합니다.

```text
tools/idols.json
```

기본 파일은 빈 배열입니다.

```json
[]
```

인물을 추가할 때는 아래 형식을 사용합니다.

```json
[
  {    "name": "강유찬",    "group": "A.C.E",    "agency": "기타",    "gender": "M",    "dob": "1997-12-31",    "cat": "K-idol"  }
]
```

필드 설명과 여러 명 추가 예시는 [`docs/DATA.md`](docs/DATA.md)를 참고하세요.

# 8. 데이터 검사

Node.js가 설치된 환경에서 프로젝트 루트 기준으로 실행합니다.

```bash
node tools/validate.js
```

성공 예시:

```text
Validation passed: 25 records
```

검사 항목:

- JSON 문법
- 배열 구조
- 필수 필드
- 문자열 형식
- 실제 존재하는 날짜
- 성별 값 `M` 또는 `F`
- 중복된 `name + group`
